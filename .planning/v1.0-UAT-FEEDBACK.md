# v1.0 UAT Feedback — Manual Testing Session 2026-03-24

## Setup Wizard (Phase 9 — DIST-05)

1. **Provider list mismatch** — Setup wizard's 12 providers don't match daemon's actual provider list. Must sync with `AgentConfig.providers` definitions.
2. **Missing optional setup steps** — No web search (firecrawl/exa/tavily) or gateway (Slack/Discord/Telegram) configuration in wizard. Both should be optional steps.
3. **Advanced user wizard path** — When user self-selects as advanced, wizard should offer navigation to configure more options (tools, gateway, memory, etc.).
4. **Input UX is number-based** — Wizard uses `[1-N]` number input instead of arrow keys/tabs. Bad UX for a terminal tool. Should use crossterm key navigation (arrows, enter, tab).
5. **Security defaults for newcomers** — Newcomer tier should default to highest approval threshold. They don't know what agent actions do — need approval gates on everything.

## CLI (Phase 9 — DIST-01, existing)

1. **Missing `tamux tui` and `tamux gui` subcommands** — No way to launch TUI or Electron from CLI.
2. **`tamux stats` broken** — Returns `Error: tag for enum is not valid, found 117` — deserialization error in protocol.
3. **`tamux attach` hangs** — Probably internal-only command. Should be hidden from help or documented as internal.
4. **`tamux new` requires Electron** — Only works when Electron is running. Should warn or hide from help.
5. **`tamux scrub` is internal** — Should be hidden from help or marked as internal.
6. **`tamux audit` doesn't show IDs** — Output says "Use --detail <id>" but the list doesn't display any IDs to paste. Need to show entry IDs in list output.
7. **`tamux settings` missing** — No CLI settings subcommand. Should allow advanced option configuration like TUI settings panel.

## TUI (Phase 10 — PRUX-03, PRUX-04)

1. **Concierge message clipped** — Welcome/onboarding message appears in a dedicated panel that clips downward. Should appear in the conversation thread instead, not a separate overlay panel.
2. **No tier/advancement settings** — Settings panel has no option to switch capability tier or view current tier. D-03 (bidirectional override) not accessible from TUI settings.
3. **Auth not reaching daemon** — Setup wizard writes to `config.json` but **daemon reads config from the SQLite database, NOT config.json**. The entire setup wizard config-write path is wrong. Must use IPC (ClientMessage) to set config in the daemon's DB, not write JSON files. This is the root cause of provider auth not working after setup. All references to config.json as a daemon config source must be removed.
4. **Missing feature settings** — Advanced tab in TUI settings doesn't show settings for Phase 2-10 features (heartbeat, memory consolidation, skill discovery, gateway, etc.).
