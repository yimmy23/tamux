<objective>
Build the plugin loader, SQLite persistence, PluginManager struct, and wire into the daemon's server.rs. On startup, the daemon scans `~/.tamux/plugins/*/plugin.json`, validates each manifest (size + schema + structural limits), persists metadata to SQLite, and reconciles stale records. IPC handlers for PluginList/PluginGet/PluginEnable/PluginDisable are wired into the server message dispatch.

Purpose: Completes the daemon-side plugin infrastructure. After this plan, plugins on disk are discoverable, validated, persisted, and queryable via IPC.

Output: `loader.rs` (validation), `persistence.rs` (SQLite CRUD), expanded `mod.rs` (PluginManager), updated `history.rs` (new tables), updated `server.rs` (init + handlers).
