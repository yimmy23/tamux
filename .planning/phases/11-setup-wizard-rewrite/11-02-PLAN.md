AgentSetConfigItem { key_path: "managed_execution.security_level".into(), value_json: format!("\"{}\"", selected_level) }

**Optional step: Model selection (per D-08 item 1) -- Familiar+ tier only:**
If tier is "familiar", "power_user", or "expert":
- Print "Fetching available models..."
- Send `ClientMessage::AgentFetchModels { provider_id, base_url, api_key }`.
- Recv `DaemonMessage::AgentModelsResponse { models_json }`.
- If models_json parses as a non-empty array of strings, show select_list with title "Select default model:" with allow_esc=true (skippable).
- If models_json is empty or fetch fails, show text_input with "Enter model name (or Esc to skip):" with default from the provider's auth state model.
- If user selects/enters a model, send: `AgentSetConfigItem { key_path: "model", value_json: format!("\"{}\"", model) }`.
- If user presses Esc, print "Skipped -- using default model."

**Optional step: Web search API key (per D-08 item 2) -- Familiar+ tier only:**
If tier is "familiar", "power_user", or "expert":
- Use select_list with title "Configure web search? (enables agent web browsing)":
  - "Firecrawl" -> prompt for firecrawl_api_key
  - "Exa" -> prompt for exa_api_key
  - "Tavily" -> prompt for tavily_api_key
  - "Skip" -> skip this step
- allow_esc=true (skippable).
- If user selects a provider, use text_input with masked=true to collect the API key.
- Persist via IPC:
  - `AgentSetConfigItem { key_path: "tools.web_search", value_json: "true" }` to enable the tool
  - `AgentSetConfigItem { key_path: "{key_name}", value_json: format!("\"{}\"", api_key) }` where key_name is "firecrawl_api_key", "exa_api_key", or "tavily_api_key"
- If user selects "Skip" or presses Esc, print "Skipped -- you can add web search later with `tamux setup`."

**Optional step: Gateway setup (per D-08 item 3) -- Power User/Expert tier only (per D-09):**
If tier is "power_user" or "expert":
- Use select_list with title "Configure a chat gateway? (Slack, Discord, Telegram)":
  - "Slack" -> prompt for slack_token
  - "Discord" -> prompt for discord_token
  - "Telegram" -> prompt for telegram_token
  - "Skip" -> skip this step
- allow_esc=true (skippable).
- If user selects a platform, use text_input with masked=true to collect the token.
- Persist via IPC:
  - `AgentSetConfigItem { key_path: "gateway.enabled", value_json: "true" }`
  - `AgentSetConfigItem { key_path: "gateway.{platform}_token", value_json: format!("\"{}\"", token) }` where platform is "slack", "discord", or "telegram"
- If user presses Esc or selects Skip, print "Skipped -- you can configure gateways later with `tamux setup`."

**Optional step: Data directory (per D-08 item 4) -- Familiar+ tier only:**
If tier is "familiar", "power_user", or "expert":
- Use text_input with "Data directory [{default}]:" where default is `amux_protocol::amux_data_dir().display()`.
- allow_esc=true (skippable).
- This is informational only for now -- the data directory is determined at daemon startup and cannot be changed via IPC after the fact. Just display the current value and note it.
- Print "Data stored at: {path}" (informational, no IPC write needed since daemon determines this at startup).

Run 'tamux' to start using tamux.
