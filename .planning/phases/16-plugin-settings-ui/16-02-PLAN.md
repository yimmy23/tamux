    **1. Create `frontend/src/lib/pluginStore.ts`:**

    Create a Zustand store following the existing agentStore.ts pattern. Named export: `usePluginStore`.

    ```typescript
    import { create } from "zustand";
    import { getBridge } from "./bridge";

    export type PluginInfoItem = {
      name: string;
      version: string;
      description?: string;
      author?: string;
      enabled: boolean;
      install_source: string;
      has_api: boolean;
      has_auth: boolean;
      has_commands: boolean;
      has_skills: boolean;
      endpoint_count: number;
      settings_count: number;
      installed_at: string;
      updated_at: string;
    };

    export type PluginSettingValue = {
      key: string;
      value: string;
      is_secret: boolean;
    };

    // Parsed from manifest settings_schema JSON
    export type PluginSettingField = {
      key: string;
      type: string;     // "string" | "number" | "boolean" | "select" | "secret"
      label: string;
      required: boolean;
      secret: boolean;
      default?: unknown;
      options?: string[];
      description?: string;
    };

    type PluginStoreState = {
      plugins: PluginInfoItem[];
      loading: boolean;
      error: string | null;

      // Selected plugin detail
      selectedPlugin: string | null;
      settingsSchema: PluginSettingField[];
      settingsValues: PluginSettingValue[];
      settingsLoading: boolean;

      // Test connection state
      testResult: { success: boolean; message: string } | null;
      testLoading: boolean;

      // Actions
      fetchPlugins: () => Promise<void>;
      selectPlugin: (name: string | null) => Promise<void>;
      toggleEnabled: (name: string, enabled: boolean) => Promise<void>;
      updateSetting: (pluginName: string, key: string, value: string, isSecret: boolean) => Promise<void>;
      testConnection: (name: string) => Promise<void>;
    };
    ```

    Implement store with `create<PluginStoreState>()`:

    - `fetchPlugins`: Calls `getBridge()?.pluginDaemonList()`, extracts `.plugins` array, sets `plugins` state. Falls back to empty array on error. Sets `loading` appropriately.

    - `selectPlugin(name)`: If `name` is null, clears selection. Otherwise:
      1. Calls `getBridge()?.pluginDaemonGet(name)` to get `settings_schema` JSON.
      2. Parse schema JSON into `PluginSettingField[]`: `Object.entries(JSON.parse(schema)).map(([key, field]) => ({ key, ...field }))`.
      3. Calls `getBridge()?.pluginGetSettings(name)` to get current values.
      4. Sets `selectedPlugin`, `settingsSchema`, `settingsValues`, `testResult: null`.

    - `toggleEnabled(name, enabled)`: Calls `pluginDaemonEnable` or `pluginDaemonDisable` based on `enabled`. Updates local `plugins` array optimistically (set enabled flag). Then refetch with `fetchPlugins()`.

    - `updateSetting(pluginName, key, value, isSecret)`: Calls `getBridge()?.pluginUpdateSettings(pluginName, key, value, isSecret)`. On success, update `settingsValues` locally (replace matching key or push new entry).

    - `testConnection(name)`: Sets `testLoading: true`, calls `getBridge()?.pluginTestConnection(name)`, sets `testResult` with response, `testLoading: false`.

    **2. Create `frontend/src/components/settings-panel/PluginsTab.tsx`:**

    Must follow 16-UI-SPEC.md visual/interaction contracts exactly. Use inline CSSProperties with CSS custom properties (no className, no external CSS).

    **PluginsTab component** (exported, top-level):
    - On mount (`useEffect`), call `fetchPlugins()`.
    - If `loading`, show nothing (panel stays blank briefly).
    - If `plugins.length === 0`, render empty state per UI-SPEC copywriting contract:
      - "No plugins installed" heading (fontSize 14, fontWeight 600, color `var(--text-primary)`)
      - "Run `tamux plugin add <name>` to install a plugin, then configure it here." body (fontSize 12, color `var(--text-muted)`)
      - Center both vertically and horizontally in the tab content area.
    - If plugins exist, render vertical list of PluginCard components separated by 1px `var(--border)` dividers.

    **PluginCard component** (internal):
    - Props: `plugin: PluginInfoItem`
    - State: `expanded: boolean` (default false)
    - Collapsed view: single row, flex, space-between.
      - Left side: plugin name (fontSize 14, fontWeight 600, color `var(--text-primary)`) + version (fontSize 12, color `var(--text-muted)`, marginLeft 8) + AuthStatusBadge (if `plugin.has_auth`).
      - Right side: `Toggle` component bound to `plugin.enabled`, onChange calls `toggleEnabled`.
      - Disabled plugins: name opacity 0.6.
    - Click anywhere on the row (except toggle) toggles `expanded`.
    - Expanded view (below collapsed row, no animation per UI-SPEC): render:
      1. Description (if present): fontSize 12, color `var(--text-secondary)`, lineHeight 1.5, paddingTop 8.
      2. `PluginSettingsForm` (if `settingsSchema.length > 0` and this is the selected plugin).
      3. "Test Connection" button (if `plugin.has_api`): `smallBtnStyle`, onClick calls `testConnection`.
         - Loading state: text "Testing...", button disabled.
         - Result: inline text below button, color `var(--success)` for success or `var(--danger)` for failure. Auto-clear success after 5 seconds.
      4. "Connect" / "Reconnect" button (if `plugin.has_auth`): `smallBtnStyle`. Text depends on auth status.
         - Per D-07: onClick opens system browser via `window.open()` (Electron will intercept via shell.openExternal if needed).
         - Per D-09: "Reconnect" triggers same flow as "Connect".
      5. Metadata footer: install_source + installed_at (fontSize 12, color `var(--text-muted)`, paddingTop 12).
    - When card expands and is not yet the selectedPlugin, call `selectPlugin(plugin.name)` to fetch schema and values.

    **PluginSettingsForm component** (internal):
    - Props: `pluginName: string, schema: PluginSettingField[], values: PluginSettingValue[]`
    - Renders fields in declaration order (schema array order) per D-03.
    - For each field, use a `SettingRow` with label + required marker:
      - Label text: `field.label` + ` *` if `field.required` (asterisk per UI-SPEC, with `title="required"`).
    - Field type to control mapping per D-04 and UI-SPEC Interaction Contracts:
      - `"string"` -> `TextInput` with value from `values` or `field.default` or `""`.
      - `"number"` -> `NumberInput` with value parsed from `values` or `field.default` or `0`.
      - `"boolean"` -> `Toggle` with value from `values` (parse "true"/"false") or `field.default` or `false`.
      - `"select"` -> `SelectInput` with `field.options ?? []`, value from `values` or `field.default` or first option.
      - `"secret"` -> `PasswordInput` with value from `values` or `""`.
    - Save on blur per D-06: each control's onChange calls `updateSetting(pluginName, field.key, stringValue, field.secret)`.
      - For TextInput, PasswordInput: fire on blur, not on every keystroke. Wrap in local state with an onBlur handler.
      - For Toggle, Select: fire immediately on change (no blur concept).
      - For NumberInput: fire on blur.
    - Field description rendered below control if `field.description` present: fontSize 12, color `var(--text-muted)`.
    - Validation per D-05:
      - Required fields: show "This field is required" error in `var(--danger)` below the control on blur if empty.
      - Number fields: show "Enter a valid number" if value doesn't parse.

    **AuthStatusBadge component** (internal):
    - Props: `hasAuth: boolean, authStatus?: "not_configured" | "connected" | "expired"`
    - If `!hasAuth`, return null.
    - Render inline: 8px dot + text label, fontSize 12, gap 4px, matching StatusIndicator pattern.
    - Dot colors per UI-SPEC: not_configured = `var(--text-muted)`, connected = `var(--success)`, expired = `var(--warning)`.
    - Text: "Not configured" / "Connected" / "Expired -- Reconnect" per copywriting contract.
    - Note: For Phase 16 the auth status is always "not_configured" since OAuth flow is Phase 18. Render this as the default.
