Create `scripts/install.sh` -- per D-10, D-11, D-13:

Script structure:
```bash
#!/bin/sh
# install.sh -- Install tamux binaries to ~/.local/bin
# Usage: curl -fsSL https://tamux.dev/install.sh | sh
#        TAMUX_VERSION=0.1.10 ./install.sh
#        TAMUX_INSTALL_DIR=/opt/tamux ./install.sh
#        ./install.sh --dry-run
set -e
```

**Variables:**
- `INSTALL_DIR="${TAMUX_INSTALL_DIR:-$HOME/.local/bin}"` -- per D-10 (~/.local/bin, no sudo)
- `BASE_URL="https://gitlab.com/api/v4/projects/PROJECT_ID/packages/generic/tamux"` -- placeholder PROJECT_ID, same as npm package
- `BINARIES="tamux-daemon tamux tamux-tui"` -- the three binaries per D-11
- `DRY_RUN=false` -- set to true when `--dry-run` flag is passed

**Argument parsing (before any function calls):**
```bash
DRY_RUN=false
for arg in "$@"; do
  case "$arg" in
    --dry-run) DRY_RUN=true ;;
  esac
done
```

**detect_platform() function:**
- `OS="$(uname -s)"`
- `ARCH="$(uname -m)"`
- Case statement for OS: `Linux` -> `linux`, `Darwin` -> `darwin`, `*` -> error "Unsupported OS: $OS"
- Case statement for ARCH: `x86_64|amd64` -> `x64`, `aarch64|arm64` -> `arm64`, `*` -> error "Unsupported architecture: $ARCH"
- `TARGET="${OS_NAME}-${ARCH_NAME}"`
- Print: `"Detected platform: ${TARGET}"`

**get_latest_version() function:**
- If `TAMUX_VERSION` env var is set, use it directly
- Otherwise, attempt to fetch latest tag from GitLab API: `curl -fsSL "https://gitlab.com/api/v4/projects/PROJECT_ID/releases" | grep -o '"tag_name":"v[^"]*"' | head -1 | grep -o '[0-9][0-9.]*'`
- If API call fails, print error "Could not determine latest version. Set TAMUX_VERSION=x.y.z" and exit 1
- `VERSION` variable set from result

**--dry-run early exit (after detect_platform and get_latest_version):**
```bash
if [ "$DRY_RUN" = true ]; then
  TARBALL="tamux-binaries-v${VERSION}-${TARGET}.tar.gz"
  echo "Platform: ${TARGET}"
  echo "Version: ${VERSION}"
  echo "Download URL: ${BASE_URL}/${VERSION}/${TARBALL}"
  echo "Install directory: ${INSTALL_DIR}"
  echo "Dry run complete -- no files downloaded or modified."
  exit 0
fi
```

**download_and_verify() function:**
- `TARBALL="tamux-binaries-v${VERSION}-${TARGET}.tar.gz"`
- `SUMS="SHA256SUMS-${TARGET}.txt"`
- `TMP_DIR="$(mktemp -d)"`
- Print: `"Downloading tamux v${VERSION} for ${TARGET}..."`
- `curl -fsSL "${BASE_URL}/${VERSION}/${TARBALL}" -o "${TMP_DIR}/${TARBALL}"`
- `curl -fsSL "${BASE_URL}/${VERSION}/${SUMS}" -o "${TMP_DIR}/${SUMS}"`
- SHA256 verification per D-13:
  ```
  if command -v sha256sum >/dev/null 2>&1; then
    (cd "$TMP_DIR" && sha256sum -c "$SUMS")
  elif command -v shasum >/dev/null 2>&1; then
    (cd "$TMP_DIR" && shasum -a 256 -c "$SUMS")
  else
    echo "Warning: sha256sum not found, skipping checksum verification"
  fi
  ```

**install_binaries() function:**
- `mkdir -p "$INSTALL_DIR"`
- `tar xzf "${TMP_DIR}/${TARBALL}" -C "$INSTALL_DIR"`
- Set executable permissions: `chmod +x "$INSTALL_DIR/tamux-daemon" "$INSTALL_DIR/tamux" "$INSTALL_DIR/tamux-tui"`
- Print: `"Installed: tamux-daemon, tamux, tamux-tui -> $INSTALL_DIR"`

**offer_desktop_app() function -- per D-11:**
- Only run if stdin is a terminal (`[ -t 0 ]`)
- Print: `"Install desktop app? [y/N] "`
- Read user input
- If "y" or "Y":
  - Determine desktop artifact name based on platform (AppImage for linux, dmg for darwin)
  - Download `tamux-desktop-v${VERSION}-${TARGET}.{ext}` from same BASE_URL
  - For Linux: save to `$INSTALL_DIR/tamux-desktop.AppImage`, chmod +x
  - For macOS: save to `/tmp/tamux-desktop.dmg`, print "Open /tmp/tamux-desktop.dmg to install"
- If non-interactive (piped), skip silently

**ensure_path() function -- per D-10, Pitfall 5:**
- Check if INSTALL_DIR already in PATH: `case ":$PATH:" in *":$INSTALL_DIR:"*) return ;; esac`
- For each existing profile file in `"$HOME/.bashrc" "$HOME/.zshrc" "$HOME/.profile" "$HOME/.bash_profile"`:
  - If file exists, append:
    ```
    echo "" >> "$profile"
    echo "# Added by tamux installer" >> "$profile"
    echo 'export PATH="'"$INSTALL_DIR"':$PATH"' >> "$profile"
    ```
- Print: `"Added $INSTALL_DIR to PATH in shell profiles."`
- Print: `"Run 'source ~/.bashrc' or open a new terminal to start using tamux."`

**Main execution flow:**
```
detect_platform
get_latest_version

# --dry-run: print detection results and exit without downloading
if [ "$DRY_RUN" = true ]; then
  # (dry-run block shown above)
fi

download_and_verify
install_binaries
offer_desktop_app
ensure_path
rm -rf "$TMP_DIR"
echo ""
echo "tamux installed successfully! Run 'tamux' to get started."
```

**Cleanup on error:** Use `trap 'rm -rf "$TMP_DIR" 2>/dev/null' EXIT` at the top after TMP_DIR is created.

Create `scripts/install.ps1` -- per D-12, D-13, Pitfall 8:

```powershell
#Requires -RunAsAdministrator
# install.ps1 -- Install tamux binaries to C:\Program Files\tamux
# Usage: irm https://tamux.dev/install.ps1 | iex
#        $env:TAMUX_VERSION = "0.1.10"; .\install.ps1
param(
    [switch]$DryRun
)

$ErrorActionPreference = "Stop"
```

**Variables:**
- `$InstallDir = "C:\Program Files\tamux"` -- per D-12
- `$BaseUrl = "https://gitlab.com/api/v4/projects/PROJECT_ID/packages/generic/tamux"` -- same placeholder
- `$Binaries = @("tamux-daemon.exe", "tamux.exe", "tamux-tui.exe")`

**Detect-Platform function:**
- `$Arch = [System.Runtime.InteropServices.RuntimeInformation]::OSArchitecture`
- Map: `X64` -> `x64`, `Arm64` -> `arm64`, else error "Unsupported architecture"
- `$Target = "windows-$Arch"` (only windows-x64 currently per D-06)
- Write-Host "Detected platform: $Target"

**Get-LatestVersion function:**
- If `$env:TAMUX_VERSION` is set, use it
- Otherwise, query GitLab API for latest release tag
- Parse and return version string

**Download-AndVerify function:**
- `$Tarball = "tamux-binaries-v$Version-$Target.tar.gz"`
- `$Sums = "SHA256SUMS-$Target.txt"`
- `$TmpDir = Join-Path $env:TEMP "tamux-install"`
- `New-Item -ItemType Directory -Force -Path $TmpDir`
- Download with `Invoke-WebRequest -Uri "$BaseUrl/$Version/$Tarball" -OutFile "$TmpDir\$Tarball"` (use `-UseBasicParsing` for older PS)
- Download checksums file
- SHA256 verification per D-13:
  ```powershell
  $ExpectedHash = (Get-Content "$TmpDir\$Sums" | Select-String $Tarball).ToString().Split(" ")[0]
  $ActualHash = (Get-FileHash "$TmpDir\$Tarball" -Algorithm SHA256).Hash.ToLower()
  if ($ActualHash -ne $ExpectedHash) {
      Write-Warning "SHA256 checksum mismatch! Expected: $ExpectedHash, Got: $ActualHash"
      throw "Checksum verification failed"
  }
  Write-Host "SHA256 checksum verified."
  ```

**Install-Binaries function:**
- `New-Item -ItemType Directory -Force -Path $InstallDir`
- Extract: `tar xzf "$TmpDir\$Tarball" -C $InstallDir` (tar available on Windows 10+)
- Verify binaries exist after extraction

**Update-Path function -- per D-12, Pitfall 8:**
- Get current system PATH: `$CurrentPath = [Environment]::GetEnvironmentVariable("Path", "Machine")`
- Check if `$InstallDir` already in PATH
- If not:
  - Update registry (persistent): `[Environment]::SetEnvironmentVariable("Path", "$CurrentPath;$InstallDir", "Machine")`
  - Update current session: `$env:Path = "$env:Path;$InstallDir"`
  - Write-Host "Added $InstallDir to system PATH."
  - Write-Host "Note: Other open terminals need to be restarted to pick up PATH change."

**Main flow:**
```powershell
Detect-Platform
$Version = Get-LatestVersion
if ($DryRun) {
    Write-Host "Would download: $BaseUrl/$Version/tamux-binaries-v$Version-$Target.tar.gz"
    Write-Host "Would install to: $InstallDir"
    exit 0
}
Download-AndVerify
Install-Binaries
Update-Path
Remove-Item -Recurse -Force $TmpDir -ErrorAction SilentlyContinue
Write-Host ""
Write-Host "tamux installed successfully! Run 'tamux' to get started."
```

Include `-DryRun` switch for testing without actual downloads.
