**npm-package/install.js** -- per D-01, D-03, D-08, D-13:
- `"use strict";` CommonJS module (no ESM -- must work with Node 18+)
- Import only Node builtins: `https`, `fs`, `path`, `os`, `crypto`, `child_process`, `zlib`
- Read VERSION from `./package.json` version field
- Define PLATFORM_MAP: `{"linux-x64": "linux-x64", "linux-arm64": "linux-arm64", "darwin-arm64": "darwin-arm64", "darwin-x64": "darwin-x64", "win32-x64": "windows-x64"}`
- Construct platform key as `${os.platform()}-${os.arch()}`
- If platform not in PLATFORM_MAP, `console.warn("tamux: unsupported platform " + key + ", skipping binary download")` and `process.exit(0)` (don't fail npm install)
- BIN_DIR = `path.join(__dirname, "bin")`
- BASE_URL = `https://gitlab.com/api/v4/projects/PROJECT_ID/packages/generic/tamux/${VERSION}` (use placeholder PROJECT_ID -- will be set during CI publish)
- TARBALL filename = `tamux-binaries-v${VERSION}-${target}.tar.gz`
- SHA256SUMS filename = `SHA256SUMS-${target}.txt`
- Implement `download(url)` function returning a Promise that follows HTTP 301/302 redirects (up to 5), returns Buffer. Use `https.get()` with redirect handling.
- Implement `verifyChecksum(filePath, expectedHash)` using `crypto.createHash("sha256")`, read file as stream, compare hex digest.
- Main flow:
  1. `fs.mkdirSync(BIN_DIR, { recursive: true })`
  2. Download SHA256SUMS file, parse to get expected hash for tarball
  3. Download tarball to `path.join(os.tmpdir(), TARBALL)`
  4. Verify SHA256 checksum -- if mismatch, warn and exit 0 (don't block install)
  5. Extract tarball: on Unix use `child_process.execFileSync("tar", ["xzf", tmpPath, "-C", BIN_DIR])`, on win32 use `child_process.execFileSync("tar", ["xzf", tmpPath, "-C", BIN_DIR])` (tar available on Windows 10+). Use execFileSync (not execSync) to avoid shell injection.
  6. Set executable permissions: `fs.chmodSync(path, 0o755)` for each binary (tamux-daemon, tamux, tamux-tui) -- skip on win32
  7. Clean up temp file
- Wrap entire main() in `.catch()` that logs warning and exits 0: `"tamux: postinstall binary download failed: " + err.message` + `"tamux: binaries will be downloaded on first run"` + `process.exit(0)`
- Add `--dry-run` flag support: if `process.argv.includes("--dry-run")`, print the download URL and platform detection result, then exit 0 without downloading. This enables testing.

**npm-package/bin/tamux.js** -- per D-04 (npx tamux starts daemon + TUI):
- `#!/usr/bin/env node` shebang line
- `"use strict";`
- Determine binary name: `process.platform === "win32" ? "tamux.exe" : "tamux"`
- Look for the native binary in `path.join(__dirname, binaryName)` (installed by postinstall into bin/)
- If binary not found:
  - Print `"tamux: binary not found, attempting download..."`
  - `require("../install")` -- this triggers the postinstall download logic as fallback
  - After require completes, re-check for binary existence with `fs.existsSync()`
  - If still not found, print `"tamux: could not download binary for your platform. Visit https://gitlab.com/anthropic/tamux/-/releases for manual download."` and `process.exit(1)`
- If binary found:
  - Import `child_process` and `spawn` the binary: `child_process.spawn(binaryPath, process.argv.slice(2), { stdio: "inherit" })`
  - Forward exit code: `child.on("close", (code) => process.exit(code ?? 1))`
  - Handle SIGINT: `process.on("SIGINT", () => child.kill("SIGINT"))`
  - Handle SIGTERM: `process.on("SIGTERM", () => child.kill("SIGTERM"))`
- When invoked with no arguments (`process.argv.length <= 2`), the native `tamux` binary receives no subcommand. This is where first-run detection (Plan 04) will intercept. For now, just forward all args as-is.
