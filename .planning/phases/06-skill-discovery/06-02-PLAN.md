       - Check remaining budget (10 seconds for LLM)
       - Check circuit breaker
       - Query skill_variants for status="draft" (add `list_skill_variants_by_status()` to history.rs)

       - Read the SKILL.md content from disk

       - Call `self.history.update_skill_variant_status(variant_id, "testing")`
       - Record provenance event
       - Return count of tested skills

         * "proven" -> "promoted_to_canonical" when success_count >= config.skill_promotion.proven_to_canonical
       - When promoting "proven" -> canonical: move SKILL.md file from current location to the appropriate non-draft location if needed
       - Call update_skill_variant_status for each promotion
       - Record provenance event for each promotion (per D-07)
       - Process all eligible variants (this is cheap -- no LLM calls)
       - Return count of promotions
