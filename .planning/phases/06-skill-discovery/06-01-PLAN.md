    3. Implement the pure functions per D-01 and RESEARCH.md Pattern 2:

       a) `pub(super) fn meets_complexity_threshold(tool_count: usize, replan_count: u32, quality_score: Option<f64>, outcome: &str, config: &SkillDiscoveryConfig) -> bool`
          - Return false if outcome != "success"
          - tool_gate = tool_count > config.min_tool_count
          - replan_gate = replan_count >= config.min_replan_count
          - quality_gate = quality_score.map_or(false, |q| q > config.min_quality_score)
          - Return tool_gate && (replan_gate || quality_gate)

       b) `pub(super) fn jaccard_similarity(a: &[String], b: &[String]) -> f64`
          - Both empty => 1.0
          - Build HashSets, compute |intersection| / |union|

       c) `pub(super) fn is_novel_sequence(tool_sequence: &[String], task_type: &str, patterns: &[&super::learning::patterns::ToolPattern], similarity_threshold: f64) -> bool`
          - Takes pre-fetched patterns (not PatternStore directly, for testability)
          - For each pattern, compute jaccard_similarity between pattern.tool_sequence and tool_sequence
          - If any similarity >= similarity_threshold, return false (not novel)
          - Otherwise return true

       d) `pub(super) fn extract_tool_sequence_from_json(json: Option<&str>) -> Vec<String>`
          - Parse JSON array of strings, return empty vec on None or parse failure
