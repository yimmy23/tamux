<objective>
Wire health monitoring, exponential backoff, and thread context tracking into the gateway poll cycle. Extend GatewayState with per-platform health, make poll functions return Result for error tracking, emit GatewayStatus events on transitions, emit HeartbeatDigest on connect/disconnect transitions, and extract thread metadata from incoming messages.

Purpose: This makes the gateway resilient to temporary API outages (auto-reconnect with backoff), gives users visibility into connection status (GatewayStatus events and HeartbeatDigest notifications), and captures thread context needed for Plan 03's reply routing.
Output: Health-aware gateway poll cycle with backoff, thread-annotated IncomingMessage, env var token fallback, per-platform configurable poll intervals, HeartbeatDigest on health transitions.
</objective>

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum GatewayConnectionStatus {
    Connected,
    Disconnected,
    Error,
}

#[derive(Debug, Clone)]
pub struct PlatformHealthState {
    pub status: GatewayConnectionStatus,
    pub last_success_at: Option<u64>,
    pub last_error_at: Option<u64>,
    pub consecutive_failure_count: u32,
    pub last_error: Option<String>,
    pub current_backoff_secs: u64,
}

impl PlatformHealthState {
    pub fn new() -> Self;
    pub fn on_success(&mut self, now_ms: u64);
    pub fn on_failure(&mut self, now_ms: u64, error: String);
    pub fn should_retry(&self, now_ms: u64) -> bool;
    pub fn status_changed(&self, old: GatewayConnectionStatus) -> bool; // old != self.status
}
