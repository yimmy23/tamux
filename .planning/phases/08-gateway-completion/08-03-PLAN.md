    1. Lock gateway_state, call `gw.discord_rate_limiter.try_acquire()`. Sleep if needed.
    2. Chunk: `gateway_format::chunk_message(&message, gateway_format::DISCORD_MAX_CHARS)`.
    3. Send each chunk as a separate API call (POST to `channels/{channel}/messages`).

    **For `send_telegram_message`:**
    1. Lock gateway_state, call `gw.telegram_rate_limiter.try_acquire()`. Sleep if needed.
    2. Chunk: `gateway_format::chunk_message(&message, gateway_format::TELEGRAM_MAX_CHARS)`.
    3. Send each chunk as a separate API call (POST to `sendMessage`).

    **After a successful send in all three functions** (any chunk), update `gw.last_response_at`:
    ```rust
    let now_ms = std::time::SystemTime::now()

    gw.last_response_at.insert(format!("{Platform}:{channel}"), now_ms);
    ```

    Replace the Phase 2 stub of `check_unreplied_messages` (currently returns `items_found: 0`) with real unreplied detection per GATE-06.

    **Strategy:**
    Use `last_incoming_at` and `last_response_at` in GatewayState (both populated by Plan 02 and Task 1 above) to detect channels where the agent received a message but hasn't responded.

    In `check_unreplied_messages`:
    a. Read `gateway_threads` to get all active gateway conversations.
    b. Read `gateway_state` to get `last_incoming_at` and `last_response_at`.
    c. Get current time `now_ms`.
    d. For each channel key in `last_incoming_at`:
       - Check if `last_response_at` for that channel is BEFORE the incoming message timestamp (or missing).
       - Check if the incoming message is older than `threshold_hours` (prevents flagging messages that just arrived).
       - If both conditions true, this is an unreplied message.
    e. Count unreplied conversations.
    f. Return `HeartbeatCheckResult` with:
       - `items_found` = number of unreplied conversations
       - `details` listing each platform:channel, sender (from gateway_threads if available), and elapsed time since last incoming message

    Make the threshold_hours parameter actually functional: only flag messages older than `threshold_hours * 3600 * 1000` milliseconds.
