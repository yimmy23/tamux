<objective>
Wire activity-aware scheduling into the heartbeat loop, priority-based check gating into the heartbeat orchestration, and the feedback-to-weight update cycle that makes the agent learn what to check.

Purpose: Make the heartbeat feel alive by running more during the operator's active hours and less during inactive periods (BEAT-06). Make check frequency adapt to what the operator actually cares about (BEAT-07). Wire the dismiss action through the IPC layer so user feedback reaches the learning system. Close the feedback loop by updating learned weights from dismissal/inaction signals after each heartbeat cycle (D-04).

Output: gateway_loop.rs with frequency modulation, heartbeat.rs with priority gating and weight-update loop, engine.rs with learned_check_weights, server.rs with AuditDismiss handler, operator_model.rs with EMA update wiring.
</objective>

       ```rust
       // --- Weight Update Loop: Learn what the operator cares about (D-04, BEAT-07) ---
       // Query feedback signals from the last 7 days (168 hours)
       let seven_days_ago = (std::time::SystemTime::now()
           .duration_since(std::time::UNIX_EPOCH)
           .unwrap_or_default()
           .as_secs() as i64) - (7 * 24 * 3600);

       let history = &self.history;
       let dismissals = history.count_dismissals_by_type(seven_days_ago).await.unwrap_or_default();
       let shown = history.count_shown_by_type(seven_days_ago).await.unwrap_or_default();
       let acted_on = history.count_acted_on_by_type(seven_days_ago).await.unwrap_or_default();

       // Compute updated priority weight for each built-in check type
       let check_types = [
           (HeartbeatCheckType::StaleTodos, "heartbeat_stale_todos"),
           (HeartbeatCheckType::StuckGoals, "heartbeat_stuck_goals"),
           (HeartbeatCheckType::UnrepliedMessages, "heartbeat_unreplied_messages"),
           (HeartbeatCheckType::RepoChanges, "heartbeat_repo_changes"),
       ];

       let decay_rate = 0.05;    // 5% penalty per dismissal (Claude's discretion)
       let recovery_rate = 0.1;  // 10% recovery per acted-on entry (Claude's discretion)

       let mut new_weights = HashMap::new();
       for (check_type, action_type_key) in &check_types {
           let dismiss_count = dismissals.get(*action_type_key).copied().unwrap_or(0);
           let total_shown = shown.get(*action_type_key).copied().unwrap_or(0);
           let recovery_count = acted_on.get(*action_type_key).copied().unwrap_or(0);
           // Inaction = shown but neither dismissed nor acted upon
           let inaction_count = total_shown.saturating_sub(dismiss_count).saturating_sub(recovery_count);

           let weight = compute_check_priority(
               dismiss_count,
               inaction_count,
               total_shown,
               recovery_count,
               decay_rate,
               recovery_rate,
           );
           new_weights.insert(check_type.clone(), weight);
       }

       // Store learned weights on AgentEngine (these are used by should_run_check gating above)
       {
           let mut weights = self.learned_check_weights.write().await;
           *weights = new_weights;
       }

       tracing::debug!(
           weights = ?self.learned_check_weights.read().await,
           "updated learned check priority weights from feedback signals"
       );
       ```

       IMPORTANT: This step completes the D-04 feedback cycle. The flow is:
       (a) User dismisses audit entries -> server handler writes `user_action = 'dismissed'` to DB
       (b) After heartbeat execution, query DB for dismissal/shown/acted-on counts
       (c) Call `compute_check_priority()` per check type to compute new weight
       (d) Store weights in `learned_check_weights` HashMap on AgentEngine
       (e) Next heartbeat cycle, `should_run_check` reads learned weights for gating
