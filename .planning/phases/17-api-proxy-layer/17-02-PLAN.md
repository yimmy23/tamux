    1. Add to `crates/amux-daemon/src/plugin/api_proxy.rs` (per D-11, APRX-01, APRX-03):
       - `pub async fn execute_request(http_client: &reqwest::Client, rendered: &RenderedRequest) -> Result<serde_json::Value, PluginApiError>`:
         - Build reqwest::Request from RenderedRequest (method, url, headers, optional body)
         - Wrap in `tokio::time::timeout(Duration::from_secs(30), ...)` for HTTP timeout
         - On timeout -> return `PluginApiError::Timeout`
         - On HTTP error status -> read body text (truncate to 2000 chars), return `PluginApiError::HttpError { status, body }`
         - On success -> parse response body as JSON. If not valid JSON, wrap raw text in `{"text": "..."}` object
         - On upstream 429 -> return `PluginApiError::RateLimited` with retry_after from header or default 60s
         - Return the parsed JSON value for response template rendering

    2. Add to `crates/amux-daemon/src/plugin/mod.rs`:
       - New fields on PluginManager: `http_client: reqwest::Client`, `rate_limiters: tokio::sync::Mutex<rate_limiter::RateLimiterMap>`, `template_registry: handlebars::Handlebars<'static>`
       - Update `PluginManager::new()` to initialize: `http_client` via `reqwest::Client::builder().timeout(Duration::from_secs(30)).build().unwrap_or_default()`, `rate_limiters` via `Mutex::new(RateLimiterMap::new())`, `template_registry` via `template::create_registry()`
       - `pub async fn api_call(&self, plugin_name: &str, endpoint_name: &str, params: serde_json::Value) -> Result<String, PluginApiError>` implementing the full flow per D-11:
         a. Lock `self.plugins` read, find plugin by name -> PluginNotFound if missing
         b. Check plugin enabled state from persistence -> PluginDisabled if disabled
         c. Get api section from manifest -> EndpointNotFound if no API or endpoint
         d. Check rate limit: lock rate_limiters, call check() with plugin RPM (or DEFAULT_REQUESTS_PER_MINUTE) -> RateLimited error with retry_after_secs=60
         e. Get settings from persistence (raw decoded values for template context, per D-04)
         f. Build template context via template::build_context(params, settings)
         g. Render request via template::render_request(&self.template_registry, api, endpoint, &context)
         h. SSRF validate via ssrf::validate_url(&rendered.url, false) -- allow_local=false for now (D-08 allow_local deferred to install-time flag)
         i. Execute HTTP via api_proxy::execute_request(&self.http_client, &rendered)
         j. Render response via template::render_response(&self.template_registry, endpoint, &response_json)
         k. Return rendered text

       - Helper method `async fn check_plugin_enabled(&self, name: &str) -> Result<bool, PluginApiError>` that queries persistence for enabled state

    Note: The `get_settings` call in the api_call flow must use `self.persistence.get_settings()` directly (NOT `self.get_settings()` which masks secrets). The template context needs raw decoded values for API key injection in request headers.
