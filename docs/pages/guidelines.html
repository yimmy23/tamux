Guidelines

Guidelines are local Markdown playbooks that sit above skills. They route agents toward the right workflow, checks, and recommended skills for a class of task.

How Guidelines Work

A guideline does not replace a skill and does not execute anything by itself. It tells the agent what kind of task it is handling, which skills are likely relevant, what order to follow, and which failure modes to consider before work starts.

Agents are prompted to consult guidelines before skill discovery. The normal flow is discover_guidelines, read_guideline, then discover_skills and read_skill when the selected guideline points to a detailed skill workflow.

Packaged installation and app startup copy bundled guideline defaults into this directory without overwriting user-created files.

CLI Commands

# Install a custom guideline
tamux guideline install ./my-guideline.md

# Install under a different filename
tamux guideline install ./my-guideline.md --name team-coding.md

# Overwrite an existing guideline
tamux guideline install ./my-guideline.md --force

# Top-level install form
tamux install guideline ./my-guideline.md

# List installed guidelines
tamux guideline list
tamux guidelines list --json

The installer accepts Markdown files only and refuses to overwrite by default.

File Format

---
name: coding-task
description: Use before implementing a feature, behavior change, or bug fix in code.
recommended_skills:
  - test-driven-development
  - systematic-debugging
  - verification-before-completion
---

# Coding Task Guideline

## Workflow

1. Inspect the current implementation.
2. Build a risk and test matrix.
3. Use test-driven development unless the user asks for another technique.

## Quality Gate

Do not call the task complete until meaningful verification has run.
FieldPurpose
nameStable lookup name used by read_guideline and discovery output.
descriptionOne-sentence trigger description used for listing and ranking.
recommended_skillsSkill names that the guideline may route the agent toward.

When To Create One

Create a new guideline when a class of work needs a repeatable task-level policy broader than a single skill, such as team-specific release rules, support process, security review expectations, data handling policy, or project-specific coding workflow.

Do not create a guideline for one-off facts, temporary task notes, or detailed tool instructions that belong in a skill.
