Memory & Security

These topics now live on separate pages so the docs can explain both more clearly.

The split

Memory and security now have dedicated pages in the GitHub Pages docs so each topic can be explained in more depth without flattening one into the other.

Memory Architecture

Layered memory, write pipeline, provenance, recall, skills, and what belongs in durable memory.

Security

Control-plane security, approvals, critique, containment, provenance, and operator-safe autonomy.

Why split them

Memory and security are tightly related, but they solve different problems. Memory is about continuity, recall, curation, and durable knowledge. Security is about transitions, approvals, containment, critique, and trust boundaries. Keeping them separate makes both easier to understand.

Where to go next

  • Read Memory Architecture for the layered model and write pipeline.
  • Read Security for the broader control-plane model.
  • Read Governance for detailed approval and structured-verdict semantics.
