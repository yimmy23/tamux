## Overview

Operational processes should be repeatable, measurable, and continuously improving.

## Workflow

1. Document the current process before proposing changes.
2. Identify bottlenecks, error-prone steps, and manual handoffs.
3. Define the desired outcome and success criteria.
4. Propose changes that remove friction points.
5. Test process changes in a controlled scope before rolling out broadly.
6. Document the new process with clear roles and responsibilities.
7. Measure adoption and effectiveness after implementation.

## Quality Gate

A process is ready when it is documented, tested, and measurable.
