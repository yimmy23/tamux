# Learning And Explanation Task Guideline

Explanations should meet the user at the right level of detail.

## Workflow

1. Infer the user's current goal: learn concept, debug, decide, or implement.
2. Use concrete examples from the user's code or context when available.
3. Explain the mental model before edge cases.
4. Call out assumptions and where details may vary.
5. Use diagrams, tables, or examples only when they clarify.
6. End with the next practical step or check.

## Quality Gate

Do not answer with a generic tutorial when the user asked about a specific system or decision.
