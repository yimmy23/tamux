## Overview

Medical device software requires regulatory compliance from design through deployment. This guideline covers the key standards and documentation required for market approval.

## Workflow

1. Classify the software per IEC 62304: Class A (no injury), B (non-serious), C (death/serious injury).
2. Establish a Software Development Plan covering design control, risk management, and change control.
3. Document requirements, architecture, and detailed design — these are regulatory deliverables.
4. Apply ISO 14971 risk management: identify hazards, estimate risks, implement controls, verify effectiveness.
5. Execute verification: unit, integration, and system testing. Document test results and coverage.
6. Validate the software against clinical needs in the intended use environment.
7. Prepare the regulatory submission file: 510(k) for FDA, Technical File for EU MDR.

## Quality Gate

Medical device compliance is complete when software safety class is determined, risk management file is complete, and regulatory submission documents are ready for review.
