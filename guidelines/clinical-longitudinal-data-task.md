## Overview

Clinical data is not "collected" — it's generated by healthcare operations. Labs are drawn when clinically indicated (not at fixed intervals), diagnoses are coded for billing, and patients disappear when they change providers. Curation here is about reconstructing a research-grade dataset from operational artifacts, then validating that the reconstruction is honest.

## Phase 1: Lab Values and Clinical Chemistry

### 1a. Unit Harmonization

| Lab | Common Units | Conversion |
|-------|-------|-------|
| Creatinine | mg/dL, μmol/L | 1 mg/dL = 88.4 μmol/L |
| Glucose | mg/dL, mmol/L | 1 mg/dL = 0.0555 mmol/L |
| Calcium | mg/dL, mmol/L | 1 mg/dL = 0.25 mmol/L |
| Bilirubin | mg/dL, μmol/L | 1 mg/dL = 17.1 μmol/L |
| WBC | K/μL, 10⁹/L | Identical |
| Hemoglobin | g/dL, g/L | 1 g/dL = 10 g/L |

**Rule**: Convert everything to a single unit system per analyte. Never mix units in analysis.

### 1b. Reference Range Validation

Don't blindly trust the lab's stated reference ranges — they vary by instrument, population, and time.

| Issue | Detection | Action |
|-------|-------|-------|
| Range drift over time | Plot ref range limits by date | Flag if range changed abruptly (new instrument) |
| Impossible values | Bounds check (K⁺ < 1 or > 15 mEq/L) | Remove or flag for review |
| Unit mismatch | Value > 100x expected | Check unit column; harmonize |
| Specimen type conflict | Serum vs. plasma vs. whole blood | Document; don't mix without adjustment |
| Hemolyzed samples | Hemolysis index; K⁺ spike | Flag; exclude for K⁺, LDH, AST |
| Fasting vs. non-fasting | Triglycerides, glucose | Flag fasting status if available |

### 1c. Population-Level Lab QC

```python
# Population-level outlier detection
def flag_lab_outliers(df, lab_col, n_iqr=5):
    q1, q3 = df[lab_col].quantile([0.25, 0.75])
    iqr = q3 - q1
    lower, upper = q1 - n_iqr * iqr, q3 + n_iqr * iqr
    return (df[lab_col] < lower) | (df[lab_col] > upper)

# Check impossible within-patient changes
df["delta"] = df.groupby("patient_id")[lab_col].diff()
# K+ change > 5 mEq/L in < 24h without clinical context is suspect
```

## Phase 2: Temporal Alignment

### 2a. The Irregular Sampling Problem

Clinical observations happen at clinically motivated, irregular intervals — sicker patients are measured more often.

| Strategy | When to Use | Caveat |
|-------|-------|-------|
| **Time windows** (e.g., "labs in prior 30 days") | Prediction tasks | Choose window based on clinical context |
| **Last observation carried forward** | Slow-changing labs | Biases toward normal values |
| **Interpolation** | Dense measurements | Assumes smooth change — rarely true in acute care |
| **Irregular time series models** | Complex trajectories | Requires careful validation |
| **Landmark analysis** | Time-to-event | Avoids immortal time bias |

### 2b. Temporal Leakage — The Fatal Flaw

- **Any feature computed using data AFTER the prediction time is leakage**.
- Define a clear index date (admission, diagnosis, enrollment) and compute all features relative to it.
- Check: `feature_date_col <= outcome_date_col` for all rows.

## Phase 3: Survival and Time-to-Event Data

### 3a. Required Fields

Every survival row: `time` (index → event/censor), `event` (1/0), `index_date`, `last_contact_date`.

### 3b. Censoring Audit

| Censoring Reason | Action |
|-------|-------|
| **End of study** | Standard administrative censoring — OK |
| **Lost to follow-up** | Flag. Compare to completers. Sensitivity analysis if different. |
| **Death from other cause** | Competing risk — do NOT treat as censored |
| **Treatment switch** | Censor at switch or use time-dependent covariates |
| **Patient withdrew consent** | Must censor at withdrawal date |

```python
from lifelines import KaplanMeierFitter

kmf = KaplanMeierFitter()
kmf.fit_censoring(df[time_col], df[event_col])
# Check: censoring uniform over time? Systematic censoring at specific dates = DQ issue
```

### 3c. Competing Risks

When competing event rate > 10%: use Cumulative Incidence Function (CIF) + Fine-Gray subdistribution hazard. Standard Kaplan-Meier is wrong when other events prevent the event of interest.

### 3d. Immortal Time Bias

The most common fatal flaw in clinical ML: patients who live longer have more opportunities to receive treatment. Fix with landmark analysis (start follow-up at fixed time post-index) or time-dependent covariates.

## Phase 4: EHR Phenotyping

### 4a. Diagnostic Code Quality

| Issue | Action |
|-------|-------|
| Code meets billing criteria only | Review against clinical notes |
| Rule-out code used as diagnosis | Exclude R/O codes, "history of" codes |
| Code granularity varies (72K ICD-10-CM codes) | Map to validated PheCode groups |
| Code appears once, never again | Flag as possible coding error |

```python
# Validated phenotype: ≥ 2 codes on different days
cohort = patients.groupby(["patient_id", "phecode"]).agg(
    n_codes=("diagnosis_code", "count"),
    n_days=("encounter_date", "nunique"),
)
cohort = cohort[(cohort["n_codes"] >= 2) & (cohort["n_days"] >= 2)]
```

### 4b. Medication Data Caveats

- Ordered ≠ administered ≠ taken. Use administration records.
- PRN (as-needed) medications: hard to quantify exposure.
- Dose: mg/kg vs. flat dose. Body weight must be available.
- Formulary changes: same drug under different names across time.

## Phase 5: Patient-Level Split Integrity

**The #1 rule of clinical ML: never split at the observation level.**

- Split at the PATIENT level, not the row/encounter.
- All visits for one patient → same split.
- All family members → same split.
- Time-varying predictions: temporal split (all train dates < all test dates).

```python
rng = np.random.default_rng(seed)
patient_ids = rng.permutation(df[patient_id_col].unique())
n_test = int(len(patient_ids) * 0.2)
test_patients = set(patient_ids[:n_test])
df["split"] = df[patient_id_col].apply(
    lambda pid: "test" if pid in test_patients else "train"
)
```

## Quality Gate

Clinical longitudinal data is ready when:
- All lab values are harmonized to a single unit system.
- Reference ranges are validated and drift is documented.
- No temporal leakage (feature dates ≤ outcome dates).
- Censoring patterns are audited and reported.
- Competing risks are addressed when event rate > 10%.
- Immortal time bias is documented and handled.
- Patient-level splits are enforced — no observation-level leakage.
- Missing data patterns are documented with rationale for handling.
