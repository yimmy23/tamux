## Overview

Frontend UI work requires understanding the design system, user flow, and responsive behavior before writing code.

## Workflow

1. Review the design specifications and design system before coding.
2. Identify states: default, hover, active, disabled, loading, error, empty.
3. Build responsive layouts that work across target breakpoints.
4. Ensure accessibility: keyboard navigation, screen reader support, color contrast.
5. Test across target browsers and devices.
6. Keep components focused on a single responsibility.
7. Verify performance: bundle size, render time, network requests.

## Quality Gate

Frontend work is complete when it matches the design spec, handles all states, and works across target devices.
