# Incident Response Task Guideline

Incident work prioritizes containment, evidence, and clear communication.

## Workflow

1. Establish severity, current impact, start time, affected systems, and user-visible symptoms.
2. Preserve evidence before destructive remediation.
3. Contain the issue with the smallest safe action.
4. Separate mitigation from root-cause analysis.
5. Keep a timeline of observations, actions, owners, and outcomes.
6. After recovery, define follow-up fixes, tests, monitoring, and prevention.

## Quality Gate

Do not jump to permanent fixes before impact, containment, and evidence are understood.
