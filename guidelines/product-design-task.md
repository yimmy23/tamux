## Overview

Product design balances user needs, technical feasibility, and business goals. This guideline structures the design process from problem definition to deliverable.

## Workflow

1. Understand the problem: what user need, what current friction, what success looks like.
2. Research existing patterns and user behavior before ideating.
3. Sketch multiple approaches before committing to one.
4. Prototype the core interaction, not every edge case.
5. Test the prototype with real users or stakeholders before refinement.
6. Document the design rationale alongside the final output.
7. Hand off with clear implementation notes: states, edge cases, responsive behavior, accessibility.

## Quality Gate

A design is ready when it solves the stated problem, has been tested, and the rationale is documented.
