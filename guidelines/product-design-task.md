# Product Design Task Guideline

Product design should connect user need to buildable scope.

## Workflow

1. Identify target users, core job, trigger moment, and desired outcome.
2. Define the smallest useful workflow before adding secondary features.
3. Specify states, permissions, empty cases, errors, and success signals.
4. Capture acceptance criteria in user-observable terms.
5. Separate MVP, follow-up, and intentionally excluded scope.
6. Include measurement or feedback loops when the goal is behavioral.

## Quality Gate

Do not design a feature as a list of components without describing the user workflow it enables.
